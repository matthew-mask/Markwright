# Markwright — agent guide

A themed Electron markdown viewer/editor. Live WYSIWYG via Tiptap, 16 fully-designed themes, slash commands, table editing, custom themes via drop-in folder.

## Stack

- **Electron 33** + electron-vite (main / preload / renderer split)
- **React 18** + TypeScript in the renderer
- **Tiptap 2** (ProseMirror) with `tiptap-markdown` for round-tripping
- **electron-builder** → NSIS installer for Windows
- **electron-updater** publishing to GitHub Releases (`matthew-mask/Markwright`, private)

## Layout

```
src/
  main/        Electron main process (window, IPC, file ops, themes registry, auto-update)
  preload/     contextBridge → window.markwright API
  renderer/    React app
    components/   Editor, TitleBar, ThemePicker, SlashMenu
    extensions/   SlashCommands (Tiptap)
    hooks/        useTheme
    styles/       base.css, editor.css, slash.css
    styles/themes/  one CSS file per theme + index.css that imports them
  shared/      IPC channel constants + types (used by main and preload)
build/         icon.png (1024x1024 source), icon.ico (generated by scripts/make-ico.cjs)
installer/     installer.nsh (NSIS hooks), skills/markwright.md (Claude Code skill, bundled)
bin/           mw.cmd (CLI shim, copied next to exe at install time)
scripts/       make-ico.cjs (build/icon.png → build/icon.ico)
```

## Commands

| Command | What it does |
|---|---|
| `npm run dev` | Launch Electron with HMR. Renderer hot-reloads; main/preload changes need a restart. |
| `npm run typecheck` | tsc on both `tsconfig.node.json` (main/preload) and `tsconfig.web.json` (renderer). |
| `npm run make-ico` | Regenerate `build/icon.ico` from `build/icon.png`. |
| `npm run package` | Full production build → `dist/Markwright Setup <version>.exe`. Runs `make-ico` first. |
| `npm run package:dir` | Build the unpacked app folder without producing an installer (faster for sanity checks). |

## Conventions / gotchas

- **Themes are CSS-only**, scoped via `[data-theme="<id>"]`. Each theme file overrides CSS variables defined in `base.css` / `editor.css` / `slash.css` and adds its own decorative rules. Adding a new theme means: write `src/renderer/styles/themes/<id>.css`, register the import in `themes/index.css`, and add an entry to `BUILTIN_THEMES` in `src/main/themes.ts`.
- **Content width is global**, not per-theme. Hardcoded `padding: 48px 15%` in `base.css`. Don't add `--mw-content-padding-x` overrides to themes — that's how we got bitten before.
- **Soft overlay** (`--mw-content-overlay`) is painted as a `background` on `.mw-main`, not a pseudo-element. A pseudo-element scrolls with content in a scroll container; the background stays anchored.
- **Tiptap's first emission** isn't a real edit — `Editor.tsx` passes `isInitial: true` so we don't mark the doc dirty from the markdown roundtrip.
- **Drag region:** the title bar uses `-webkit-app-region: drag` everywhere except `.mw-btn`, which is `no-drag`. Don't flip the inheritance back the other way.
- **Slash commands** roundtrip through `tiptap-markdown` only because they map to standard markdown blocks. Don't add custom Tiptap nodes (e.g. callouts) without also writing a custom serializer/parser, or the doc will round-trip incorrectly on save/reopen.
- **File association + Claude skill** are wired through `installer/installer.nsh`. The skill (`installer/skills/markwright.md`) is bundled as an `extraResources` entry, copied into `%USERPROFILE%\.claude\skills\` during install.

## Release process

The team installs from GitHub Releases; auto-update pulls from there too. Steps to ship a new version:

1. **Bump version** in `package.json` (e.g. `0.1.0` → `0.1.1`). SemVer applies — auto-update only fires when the published version is greater than the installed one.
2. **Stage and commit** the version bump along with whatever changes are shipping.
3. **Tag and push:**
   ```powershell
   git tag v0.1.1 -m "Markwright v0.1.1"
   git push origin main --tags
   ```
4. **Build the installer:**
   ```powershell
   npm run package
   ```
   Output: `dist\Markwright Setup 0.1.1.exe` plus the `.blockmap` and `latest.yml` files (those last two are required for auto-update — `latest.yml` is the feed `electron-updater` reads, `.blockmap` powers delta updates).
5. **Write release notes** to `dist\release-notes-v0.1.1.md` (gh + PowerShell + heredoc + unicode = parser pain; always use a notes file).
6. **Publish the release:**
   ```powershell
   gh release create v0.1.1 --title "Markwright v0.1.1" --notes-file dist\release-notes-v0.1.1.md `
     "dist\Markwright Setup 0.1.1.exe" `
     "dist\Markwright Setup 0.1.1.exe.blockmap" `
     "dist\latest.yml"
   ```
7. **Verify** by opening the release page and confirming all three files attached. Existing installs will auto-update on their next launch.

### Required artifacts on every release

| File | Purpose |
|---|---|
| `Markwright Setup <ver>.exe` | The installer teammates download manually for first install |
| `Markwright Setup <ver>.exe.blockmap` | Enables binary-diff auto-updates (smaller downloads) |
| `latest.yml` | The feed `electron-updater` polls — **without this, auto-update is broken** |

### What the installer does

1. Installs Markwright per-user
2. Registers as default `.md` / `.markdown` handler
3. Copies `mw.cmd` next to the exe + appends install dir to user PATH (idempotent)
4. Creates `%USERPROFILE%\.claude\skills\` and drops `markwright.md` skill into it

### Code signing — currently NOT done

The installer is unsigned. SmartScreen warns on first run; teammates click "More info" → "Run anyway". Fine for internal alpha; before broader rollout, get a code-signing cert. Once obtained, set env vars `CSC_LINK` (path to .pfx) and `CSC_KEY_PASSWORD` before `npm run package`.

## Common tasks

### Adding a new theme

1. Create `src/renderer/styles/themes/<id>.css`. Override the CSS variables defined in `base.css` / `editor.css`. Theme styles should live in this one file.
2. Add `@import './<id>.css';` to `src/renderer/styles/themes/index.css`.
3. Register in `BUILTIN_THEMES` in `src/main/themes.ts`.
4. Add a `.mw-theme-preview[data-theme="<id>"]` block at the bottom of the theme CSS so it renders correctly in the picker preview tile.

### Adding a slash command item

Edit the `ITEMS` array in `src/renderer/extensions/SlashCommands.ts`. Each item is `{ title, description, icon, keywords, run }`. The `run` callback gets `{ editor, range }`; call `editor.chain().focus().deleteRange(range)...run()` and finish with the Tiptap command that does the actual transform.

### Hot-reload limits

- **Renderer changes** (CSS, React, Tiptap config): HMR — usually instant.
- **Preload changes**: rebuild but window keeps running; sometimes needs reload.
- **Main process changes** (`src/main/**`): full restart of `npm run dev` required.
